# AITAM - AI Task Manager 要件定義書

## 1. プロジェクト概要

個人・チームのプロジェクトと進捗状況・タスクを管理するWebアプリケーション。
すべての操作をAIとの自然言語対話で行う「AI-first」な管理ツール。

**プロジェクト名:** AITAM (AI Task Manager)

---

## 2. コンセプト

- **AI対話操作** — ボタンやフォームではなく、AIに話しかけて操作する
- **モバイルファースト** — スマホでの操作を前提とし、PCでも違和感なく使える
- **シンプル** — 必要最低限の機能で素早く使い始められる

### 4.4 タスク管理

タスクは **期限ありタスク** と **期限なしタスク（状況メモ）** の2種類がある。
AI対話でタスクを登録する際、入力文の内容からどちらかを自動判断する。

| 種類 | 説明 | 例 |
|---|---|---|
| 期限ありタスク | 締切日のある通常のタスク | 「デザイン提出 4/14まで」 |
| 期限なしタスク | 現在の状況やメモを記録する | 「今クライアント返事待ち中」「素材収集中」 |

**AI自動判断の基準:**
- 日付・期限を示す表現（「〜まで」「〜日」「締切」等）→ 期限ありタスク（`due_date` を設定）
- 状況を示す表現（「今〜中」「〜待ち」等）やメモ的な内容 → 期限なしタスク（`due_date` = NULL）

### 4.5 AI対話インターフェース

| ID | 機能 | 説明 | 実装状況 |
|---|---|---|---|
| A-01 | 自然言語入力 | テキストでAIに指示を送信 | ✅ |
| A-02 | 音声入力 | Web Speech APIによる音声認識（日本語） | ✅ |
| A-03 | プロジェクト操作 | 「新しいプロジェクトを登録して」→ 作成 | ✅ |
| A-04 | タスク操作 | 「タスクの締切日を4/14に」→ 更新 | ✅ |
| A-05 | 複合操作 | 1入力で複数の操作を実行（プロジェクト作成+タスク追加等） | ✅ |
| A-06 | ページ遷移 | 「タスクを表示して」「今日のタスク」→ 画面遷移 | ✅ |
| A-07 | 設定変更 | 「完了を表示して」「カレンダー表示にして」→ 設定切替 | ✅ |

| A-09 | URL連動コンテキスト | `/tasks?project=xxx` 表示中はそのプロジェクトを暗黙の操作対象にする | ✅ |

| A-11 | 自然言語日付変換 | 「来週中」「明日」「4月15日」等をYYYY-MM-DDに変換 | ✅ |
| A-12 | AI応答エリア消去 | ×ボタンで応答エリアを閉じられる | ✅ |
| A-13 | IME対応 | 日本語変換中のEnterで送信されない | ✅ |

**AIコマンド例:**
```
「新しいプロジェクト『Webサイトリニューアル』を登録して」
「テストプロジェクト：ステータスは提案中、来週中に提出予定」  → 複合操作
「タスク『デザイン作成』の締切日を4/14に設定して」
「進行中のタスクを全部見せて」
「今日のタスク」                        → /tasks?filter=today に遷移
「今週のタスク」                        → /tasks?filter=week に遷移
「テストのタスク見せて」                → /tasks?project=xxx に遷移
「完了を表示して」                      → 設定変更
「カレンダー表示にして」                → 設定変更
「今クライアントの返事待ち」            → 期限なしタスク（状況メモ）として登録
「バナー作成 4/15まで 急ぎ」           → 期限ありタスク（優先度高）として登録
「事前うちあわせ完了にして」            → あいまい検索で「事前打ち合わせ」を特定
```

## 6. RLS（Row Level Security）

### teams
- SELECT: `team_members` に自分の所属がある
- INSERT: 認証ユーザーなら可
- UPDATE/DELETE: ownerのみ

### team_members
- SELECT: `user_id = auth.uid()`（自分の所属のみ）
- INSERT: 認証ユーザーなら可
- DELETE: ownerのみ（同じチーム内）

### projects
- SELECT: 自分が作成 or 同じチームのメンバー
- INSERT: `user_id = auth.uid()`
- UPDATE: 自分が作成 or 同じチームのメンバー
- DELETE: 自分が作成

### tasks / task_links
- CRUD: 所属プロジェクトのuser_idが自分

### 7.1 画面一覧

| 画面 | パス | 説明 |
|---|---|---|
| ログイン | `/login` | メール + パスワードログイン |
| サインアップ | `/signup` | アカウント作成（表示名付き） |
| オンボーディング | `/onboarding` | チーム選択・作成（初回ログイン時） |
| ルート | `/` | 認証チェック → ダッシュボード or ログイン or オンボーディング |
| ダッシュボード | `/dashboard` | 直近タスク5件 + サマリー + プロジェクト概要 |
| タスク一覧 | `/tasks` | 全プロジェクトのタスク（リスト or カレンダー） |
| タスク（フィルタ） | `/tasks?project=xxx` | 特定プロジェクトのタスク |
| タスク（日付） | `/tasks?filter=today` | 今日/今週のタスク |
| 設定 | `/settings` | プロフィール・チーム管理・表示設定・ログアウト |

### 7.2 レイアウト

```
┌─────────────────────────────────────┐
│  ヘッダー (AITAM + チーム名 / ユーザー名) │
├─────────────────────────────────────┤
│                                     │
│      メインコンテンツ                 │
│   (ダッシュボード / タスク / 設定)      │
│                                     │
├─────────────────────────────────────┤
│  AI応答エリア (表示時のみ)  [×]       │
├─────────────────────────────────────┤
│  AI入力バー [テキスト入力] [🎤] [送信]  │
├─────────────────────────────────────┤
│  ボトムナビ (ダッシュボード / タスク / 設定) │
└─────────────────────────────────────┘
```

- ヘッダー: アプリ名 + チーム名 / ユーザー名
- AI対話バーは画面下部に固定（モバイル操作性優先）
- AI応答エリアは送信後に表示、×ボタンで閉じる
- ボトムナビゲーション: ダッシュボード / タスク / 設定

### 7.3 ダッシュボード

- **直近タスク（上部）** — 未完了・期限ありタスクを期日順に5件表示。プロジェクト名クリックでそのプロジェクトのタスク一覧へ遷移
- **サマリーカード** — プロジェクト数 / タスク数 / 期限超過数
- **プロジェクト一覧** — 進捗バー付き。クリックでそのプロジェクトのタスク一覧へ遷移。タスクの直近期日が近い順にソート

### 7.4 タスク（リスト表示）

- プロジェクトごとにグループ表示（プロジェクト名クリックでフィルタ遷移）
- タスクは期限あり→なし、期限近い順にソート
- カード左ボーダー色: 超過=赤 / 完了=緑 / 期限あり=青 / 期限なし=グレー
- 「全て」ボタンでフィルタ解除

### 7.5 タスク（カレンダー表示）

- 月間カレンダー（月曜始まり）
- 各日にタスクをドット+タイトルで表示（最大3件、超過分は「+N件」）
- 月ナビゲーション（前月/次月/今日）
- カレンダー下部にその月のタスクを日付順でリスト表示

## 8. AI処理アーキテクチャ

### 8.1 APIルート

`POST /api/chat`
- リクエスト: `{ message, currentProjectId?, contextProjectName? }`
- レスポンス: `{ reply, navigateUrl?, contextProject?, settingsUpdate? }`

### 8.2 Function Calling（Geminiツール）

| 関数名 | 説明 |
|---|---|
| create_project | プロジェクト作成 |
| list_projects | プロジェクト一覧取得 |
| update_project | プロジェクト更新 |
| delete_project | プロジェクト削除 |
| create_task | タスク作成 |
| update_task | タスク更新 |
| delete_task | タスク削除 |
| list_tasks | タスク一覧取得 |
| add_task_link | タスクにリンク追加 |
| navigate | ページ遷移 |
| update_settings | 表示設定変更 |

### 8.3 コンテキスト管理

- **URLコンテキスト:** `/tasks?project=xxx` 表示中はそのプロジェクトが暗黙の操作対象
- **会話コンテキスト:** AI操作でプロジェクトが特定されると、以降のメッセージでも継続（`contextProjectRef`で保持）
- **優先順位:** URLコンテキスト > 会話コンテキスト

### 8.4 あいまい名前検索

プロジェクト名・タスク名の検索は以下の優先順で実行:
1. **部分一致**（DB側ILIKE `%keyword%`）
2. **文字順序一致**（各文字が順序通りに含まれるか）— 例: 「テスト２」→「テストプロジェクト２」
3. **bigram一致**（2文字組み合わせの一致数）— 例: 「事前」が含まれる
4. **共通文字数**（半数以上一致でスコア付与）

### 8.5 モデルフォールバック

1. gemini-2.5-flash-lite（メイン）
2. gemini-2.0-flash-lite（フォールバック1）
3. gemini-2.0-flash（フォールバック2）

503/429エラー時に自動で次のモデルに切替。
