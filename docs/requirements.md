# AITAM - AI Task Manager 要件定義書

## 1. プロジェクト概要

個人・チームのプロジェクトと進捗状況・タスクを管理するWebアプリケーション。
すべての操作をAIとの自然言語対話で行う「AI-first」な管理ツール。

**プロジェクト名:** AITAM (AI Task Manager)

---

## 2. コンセプト

- **AI対話操作** — ボタンやフォームではなく、AIに話しかけて操作する
- **モバイルファースト** — スマホでの操作を前提とし、PCでも違和感なく使える
- **シンプル** — 必要最低限の機能で素早く使い始められる

---

## 3. 技術スタック

| カテゴリ | 技術 |
|---|---|
| フレームワーク | Next.js (App Router) |

## 4. 機能要件

### 4.1 ユーザー管理

| ID | 機能 | 説明 |
|---|---|---|
| U-01 | サインアップ | メールアドレス + パスワードでアカウント作成 |
| U-02 | ログイン / ログアウト | メール + パスワード認証 |
| U-03 | プロフィール編集 | 表示名・アバターの変更 |
| U-04 | チームメンバー管理 | メンバーの招待・削除（将来拡張） |

### 4.2 プロジェクト管理

| ID | 機能 | 説明 |
|---|---|---|
| P-01 | プロジェクト作成 | 名前・説明・ステータスを設定して作成 |
| P-02 | プロジェクト一覧 | 全プロジェクトの一覧表示（ソート・フィルタ対応） |
| P-03 | プロジェクト詳細 | プロジェクトの情報とタスク一覧を表示 |
| P-04 | プロジェクト編集 | 名前・説明・ステータス等の更新 |
| P-05 | プロジェクト削除 | プロジェクトの削除（配下タスクも削除） |
| P-06 | 進捗率表示 | タスクの完了率からプロジェクト進捗を自動算出 |

**プロジェクトステータス:** `提案（未定）` / `提案（高）` / `制作中` / `納品済`

### 4.3 タスク管理

| ID | 機能 | 説明 |
|---|---|---|
| T-01 | タスク作成 | タイトル・説明・締切日・優先度・担当者を設定 |
| T-02 | タスク一覧 | プロジェクト内タスクの一覧表示 |
| T-03 | タスク編集 | 各フィールドの更新 |
| T-04 | タスク削除 | タスクの削除 |
| T-05 | ステータス変更 | 未着手 → 進行中 → 完了 などの状態遷移 |
| T-06 | 締切日ソート | 締切日が近い順にソート表示 |
| T-07 | 優先度設定 | 高・中・低の3段階 |
| T-08 | 関連リンク | タスクにURL（複数可）を紐付けて管理 |

**タスクステータス:** `未着手` / `進行中` / `レビュー中` / `完了`

### 4.4 AI対話インターフェース

| ID | 機能 | 説明 |
|---|---|---|
| A-01 | 自然言語入力 | テキストでAIに指示を送信 |
| A-02 | プロジェクト操作 | 「新しいプロジェクトを登録して」→ プロジェクト作成 |
| A-03 | タスク操作 | 「タスクの締切日を4/14に設定して」→ タスク更新 |
| A-04 | 一覧・検索 | 「締切が近い順にプロジェクト一覧を表示して」→ ソート表示 |
| A-05 | 確認応答 | 操作前に内容を確認し、実行結果をフィードバック |
| A-06 | コンテキスト保持 | 会話の文脈を保持し、「それ」「このタスク」等の指示語に対応 |

**AIコマンド例:**
```
「新しいプロジェクト『Webサイトリニューアル』を登録して」
「タスク『デザイン作成』の締切日を4/14に設定して」
「締切が近い順にプロジェクト一覧を表示して」
「進行中のタスクを全部見せて」
「このタスクを完了にして」
「優先度が高いタスクだけ表示して」
```

## 5. データモデル

### 5.1 users

| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | Supabase Auth UID |
| display_name | text | 表示名 |
| avatar_url | text | アバター画像URL |
| created_at | timestamptz | 作成日時 |
| updated_at | timestamptz | 更新日時 |

### 5.2 projects

| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | プロジェクトID |
| user_id | uuid (FK → users) | オーナーユーザー |
| name | text | プロジェクト名 |
| description | text | 説明 |
| status | text | ステータス（提案（未定）/提案（高）/制作中/納品済） |
| created_at | timestamptz | 作成日時 |
| updated_at | timestamptz | 更新日時 |

### 5.3 tasks

| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | タスクID |
| project_id | uuid (FK → projects) | 所属プロジェクト |
| user_id | uuid (FK → users) | 作成者 |
| assignee_id | uuid (FK → users) | 担当者 |
| title | text | タスクタイトル |
| description | text | 説明 |
| status | text | ステータス（未着手/進行中/レビュー中/完了） |
| priority | text | 優先度（高/中/低） |
| due_date | date | 締切日 |
| created_at | timestamptz | 作成日時 |
| updated_at | timestamptz | 更新日時 |

### 5.4 task_links

| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | リンクID |
| task_id | uuid (FK → tasks) | 所属タスク |
| url | text | リンクURL |
| label | text | 表示ラベル（任意） |
| created_at | timestamptz | 作成日時 |

## 6. 画面構成

### 6.1 画面一覧

| 画面 | パス | 説明 |
|---|---|---|
| ログイン | `/login` | メール + パスワードログイン |
| サインアップ | `/signup` | アカウント作成 |
| ダッシュボード | `/` | プロジェクト概要 + AI対話 |
| プロジェクト一覧 | `/projects` | プロジェクトリスト |
| プロジェクト詳細 | `/projects/[id]` | プロジェクト情報 + タスク一覧 |
| 設定 | `/settings` | プロフィール編集 |

### 6.2 レイアウト

```
┌─────────────────────────────┐
│  ヘッダー (アプリ名 + メニュー)  │
├─────────────────────────────┤
│                             │
│      メインコンテンツ         │
│   (プロジェクト/タスク表示)    │
│                             │
├─────────────────────────────┤
│  AI対話バー (常時表示)        │
│  ┌───────────────────┐ [送信] │
│  │ AIに指示を入力...    │      │
│  └───────────────────┘      │
└─────────────────────────────┘
```

- AI対話バーは画面下部に固定（モバイルでの操作性を優先）
- メインコンテンツはスクロール可能
- ボトムナビゲーションでダッシュボード / プロジェクト / 設定を切替

## 7. 非機能要件

| 項目 | 要件 |
|---|---|
| レスポンシブ | モバイル (375px~) を基準に、タブレット・PCにも対応 |
| パフォーマンス | 初回読み込み 3秒以内（Vercel Edge） |
| セキュリティ | Supabase RLS で行レベルセキュリティを実装 |
| 認証 | Supabase Auth によるセッション管理 |
| AI API | サーバーサイドでClaude APIを呼び出し（APIキーを秘匿） |
